# PRD: AI 客服/销售漏斗（App 引流营销）

## 目标
为跨境卖家/本地商家提供 24/7 私域客服与售前转化助手，核心面向 App 引流与注册转化：
打招呼 -> 介绍产品 -> 注册引导 -> 必要时转接人工。

同时覆盖：
- 产品问题咨询（知识库问答）
- 二次消费/复购引导（情绪价值 + 个性化推荐）
- 售后问题解决
- 疑难问题转人工（收集信息 + 工单 + 通知）

## 目标用户
- WhatsApp/Telegram 私域获客的商家
- 需要把聊天流量转到 App 注册/下单的商家

## MVP 成功指标
- 引流注册转化率（对话 -> 注册完成）>= 15%
- 人工承接占比 <= 40%（60% 由机器人自助解决）
- 首响 < 3 秒
- 用户情绪正向比例 >= 70%（简单规则/评分）

## MVP 范围
### 商家侧配置
- 商家工作区（多租户隔离）
- 产品资料配置（卖点/价格/活动/下载链接/注册步骤）
- 漏斗脚本配置（打招呼/介绍/注册引导/兜底话术）
- 知识库管理（Q&A、标签、启用/停用、更新时间）
- 转人工规则（关键字、置信度、负向情绪、重复失败、退款争议）
- 人工接管通知（Telegram 群）

### 用户侧对话
- 售前漏斗引导（<= 8 轮完成注册 happy path）
- 产品问题：从知识库回答，并标注引用来源（KB 条目）
- 复购引导：根据意图/阶段提供推荐与优惠
- 售后：常见问题自助排障；复杂问题转人工

### 可观测性
- 会话日志与漏斗步骤完成率
- 转人工事件与原因
- 每日摘要（新增线索、注册数、Top 问题、未解决列表）

## 非目标（MVP 不做）
- 完整 CRM 管理
- 支付/订单系统深度集成
- 语音通话
- 高级 RAG/向量库（后续迭代）

## 功能需求
- R1 多租户商家隔离
- R2 先 Telegram，WhatsApp 作为下一里程碑
- R3 漏斗状态机：greet -> intro -> register -> verify -> done
- R4 知识库问答：先关键词/标签匹配，后续再做语义检索
- R5 转人工：生成工单 + 会话摘要 + Telegram 群通知
- R6 情绪价值：同理心模板、安抚、鼓励、适度人格化
- R7 合规：避免收集不必要 PII，不输出违法/高风险建议

## 关键对话设计（MVP）
A 打招呼：识别语言/意图（购买/咨询/售后）
B 介绍：3 个卖点 + 1 个社证 + 1 个 CTA
C 注册引导：下载链接 + 分步注册；卡住则收截图并升级
D 转人工：负向情绪/3 次失败/退款争议/未知意图立即升级

## 验收标准（摘要）
- AC1 商家无需写代码即可配置漏斗与知识库
- AC2 Happy path <= 8 轮可完成注册引导
- AC3 知识库回答带引用（条目 id 或标题）
- AC4 转人工会创建结构化工单并通知 Telegram 群

## 风险
- WhatsApp 自动化合规与稳定性
- 商家知识库质量影响准确率
- 隐私与数据合规

## 里程碑
- M1 Telegram MVP（漏斗 + KB + 转人工 + 日报）
- M2 WhatsApp 接入
- M3 语义检索/RAG + 更强个性化
